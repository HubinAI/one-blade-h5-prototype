# 《我只要一刀》H5 原型 V0.2

基于 React + TypeScript + HTML5 Canvas + Vite 实现的竖屏手机 demo。当前版本聚焦 V0.2 正式单局体验：随时挥刀、刀势越满越爽、指尖实时切割、收刀爆发、10 关基础流程。

## V0.2 已实现

- 单一核心资源：刀势能量，无固定冷却和大招按钮。
- 正式数值集中在 `src/game/config/balance.ts`，包含生命、刀势、挥刀、敌军、补给和战场边界。
- 按下瞬间锁定本次刀势等级，当前刀势归零，并立即在指尖生成刀芒。
- 拖动过程中实时切割敌军，同一敌人在同一刀内只会被命中一次。
- 刀芒会随持续时间、轨迹长度、急转弯衰减，耗尽后自动收刀。
- 击杀回能改为收刀后统一结算，避免挥刀中途能量乱跳。
- 步兵、盾兵、火药兵、阵眼兵四类敌军。
- 盾兵按阶段处理：残锋挡住、常锋破盾、强锋/破阵锋直接破盾。
- 火药兵被切中后点燃，收刀时爆炸并连锁影响周围单位。
- 阵眼兵被切中后标记，收刀时按刀势阶段带动周围阵型崩散。
- 战鼓、刀魂、火油三类补给，通过刀芒切中拾取，每局最多 4 个，每波最多 1 个。
- 底部守城武将的刀、刀鞘气场、拔刀/收刀状态与刀势融合表现，90% 以上有金色气浪。
- 10 个主线关卡，4-6 波逐步引入盾兵、火药兵、阵眼兵和补给组合。
- 胜利、失败、结算、评分、localStorage 解锁进度。
- 开发调试面板：战斗中按 `D` 开关，显示 energy、阶段、剩余挥刀时间/长度、波次、敌军数和最大单刀。

## 目录说明

```text
src/
  App.tsx                  React 屏幕流：菜单、选关、战斗、结算
  game/
    Game.ts                Canvas 游戏控制器与核心战斗循环
    GameCanvas.tsx         Canvas 输入、缩放和 requestAnimationFrame
    config/balance.ts      正式单局核心数值配置
    config/                画布尺寸、兼容参数导出
    systems/               刀势、碰撞、粒子等小系统
    types.ts               游戏数据类型
  data/
    levels.ts              10 关波次与补给配置
    enemies.ts             敌军表现定义
    pickups.ts             补给表现定义
  components/              菜单、选关、结算组件
  styles/global.css        竖屏 H5 样式
  utils/math.ts            几何和随机工具
```
