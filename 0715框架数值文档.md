## 二、战斗区域定义

```
src/game/config/balance.ts → BATTLEFIELD_ZONES
```

| 区域 | y 范围 | 作用 |
|---|---:|---|
| 出生区 | -60 ~ 60 | 怪物刚出现，不做复杂表现 |
| 快速入场区 | 60 ~ 400 | 所有怪物加速进入中场，减少等待 |
| 中场变化区 | 420 ~ 560 | 特殊怪物开始体现差异 |
| 收割压力区 | 560 ~ 720 | 玩家主刀/副刀主要收割区域 |
| 防线区 | 720+ | 触线扣血 |

### 快速入场阶段（entryPhase）

- 所有基础敌人 + 精英生成时初始化
- speedMultiplier: 1.8x
- maxDuration: 1.2s
- endY: 400（y≥400 或超时后自动结束）
- 急冲兵（30%步兵）不参与 entryPhase，使用 rush_then_slow（2.2x→0.8x）

## 三、副刀槽位设计（最终版）

### 布局
- 左槽（蓄势）：圆心"斩"，色蓝 #5bc0ff，x=24
- 右槽（破点）：圆心"破"，色橙 #ff6a33，x=DESIGN_WIDTH-76
- 半径：iconR=26，y=warriorY(760)+5=765
- 底框：rgba(18,16,14,0.92) 实心圆盘

### 三种状态的视觉语言

| 状态 | 圆心 | 圆圈 | 底部标签 | 额外反馈 |
|---|---|---|---|---|
| **CD态** | 22px 900字重 "斩"/"破" | 细灰底圈(1.5px) + 彩色进度弧(4px) | "X秒"(11px,700) | 最后1秒(ratio≥0.85)：外圈多层脉冲闪光(12Hz) + 文字高亮 |
| **Ready态** | 22px 900字重 "斩"/"破" | 完整彩色弧(4px) + 脉冲双层环(1.5px,shadow 6~12) | "可" | — |
| **空状态** | "空"(10px) | 暗淡边框 | "蓄势"/"破点" | — |

### CD最后1秒闪光参数
```
条件: ratio ≥ 0.85（最后约 0.75s）
频率: 12Hz（φ = sin(elapsed * 12 + slotIndex * 3)）
外层大光环: iconR + 4 + φ*3 动态半径, lineWidth=3, shadow=10+φ*16
中层光环: iconR + 1, lineWidth=1.5, shadow=6+φ*10
填充发光底: 18%槽位色半透（globalAlpha = 0.18 * intensity）
文字: 900字重, shadowBlur=8+φ*12, 字号12px
```

## 四、能量条

### 关键修复
- `roundRect()` 函数必须始终先调用 `ctx.beginPath()`
- 缺失 beginPath 会导致与前一个未关闭路径合并，fillW 扩展到整条

### 参数
```
eBarW = 200px
eBarH = 14px
eBarX = (390-200)/2 = 95
eBarY = 844-36 = 808
fillW = eBarW * (this.energy / 100)
背景: rgba(10, 8, 6, 0.85) 深色实心
边框: rgba(255, 214, 124, 0.3)
```

### 段位色（极淡，仅视觉参考）
```
 0~10%: rgba(192, 208, 224, 0.03)
10~40%: rgba(192, 208, 224, 0.04)
40~90%: rgba(91, 192, 255, 0.04)
90~100%: rgba(255, 211, 90, 0.05)
```

### 渐变填充
```
 0~40% (轻斩): #666 → #c0d0e0
40~90% (强斩): #c0d0e0 → #5bc0ff
90~100% (破阵): #5bc0ff → #ffd35a
```

## 六、军令宝箱流程

1. 精英死亡 → 掉落宝箱（3秒倒计时）
2. 宝箱打开 → 旋转符文动画（0.8s）
3. 揭示军令 → **暂停游戏**（paused_for_chest 阶段）
4. React 弹窗显示 buff 信息（点空白/按钮关闭）
5. 关闭后 → 应用 buff，图标 0.7s 飞向左上角（暂停按钮下方）
6. 常驻 buff icon 列表（active-buff-list，暂存于暂停按钮左下）

## 八、基础架构

### Git 沟通规则
- 大海反馈默认基于 **http://localhost:5173**（本地 dev server）
- 除非特别说明在 GitHub Pages 上看
- GitHub Pages 只用于分发给别人/手机扫码

### Dev Server 缓存问题
修改源码后验证能否看到的完整流程：
```
1. 修改 src/*.ts 文件
2. curl -s "http://localhost:5173/src/.../file.ts" | grep -c "新代码关键字"
3. 如果返回 0 → rm -rf node_modules/.vite → 杀 node.exe → 重启 vite
4. 再次 curl 验证
5. Playwright 截图视觉确认
```

### Cache 控制（V0715007）
index.html 增加了：
```html
<meta http-equiv="Cache-Control" content="no-cache, no-store, must-revalidate">
<meta http-equiv="Pragma" content="no-cache">
<meta http-equiv="Expires" content="0">
```
