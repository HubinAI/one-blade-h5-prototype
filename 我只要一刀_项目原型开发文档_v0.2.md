# 《我只要一刀》项目原型开发文档 V0.2

> 面向 Codex / 前端原型开发使用。  
> V0.2 重点目标：解决 V0.1 “等冷却再画一刀”导致的单一节奏疲劳问题。  
> 核心改动：取消固定斩击冷却，改为 **刀势能量制**；玩家随时可挥刀，但刀势越满，这一刀越长、越亮、越强。  
> 原型优先级：手感即时性 > 刀势强弱传达 > 一刀边界清晰 > 连锁爽感 > 关卡目标 > 图鉴/装备占位。

---

## 0. 给 Codex 的总开发指令

请基于 **React + TypeScript + HTML5 Canvas + Vite** 开发竖屏手机原型游戏《我只要一刀》V0.2。

本版本不再采用“固定冷却 + 大招蓄能”的双进度设计，而采用一个统一资源：**刀势能量 Blade Energy**。

核心玩法：

- 敌军从屏幕上方向下压近；
- 玩家随时可以按下并拖动屏幕挥刀；
- 当前刀势能量越高，本次刀芒越长、越宽、越亮、越强；
- 玩家按下时锁定当前刀势等级，指尖立即出现刀芒；
- 拖动过程中刀芒实时切割敌军，并随着时间、距离、急转弯逐渐变弱；
- 松手或刀芒耗尽时，本次一刀结束并进入“收刀结算”；
- 收刀时统一触发火药爆炸、阵眼崩散、同阵连锁、评价播报；
- 刀势随时间自动恢复，也会因击杀、连锁、切中补给而加快恢复；
- 底部武将站在城墙上，只做蓄势、拔刀、收刀和方向跟随表现，不飞入战场；
- 玩家真正控制的是远程刀气，命中逻辑完全由指尖刀芒轨迹完成；
- 敌军触碰底部防线扣生命；
- 清完当前关卡全部波次即胜利，生命归零即失败；
- 每关结算显示评分、最大单刀击杀数、连锁数、图鉴解锁、装备掉落。

开发目标：

1. 可以在浏览器中直接运行；
2. 支持鼠标和手机触控；
3. 适配竖屏手机比例；
4. 第一版用几何图形 / 纸片兵图标即可；
5. 重点验证“随时可挥刀，但强刀更爽”的节奏是否优于“等冷却再挥刀”；
6. 重点实现刀势能量表现、实时切割、刀芒衰减、收刀连锁、底部武将同步表现。

## 1. 项目定位

### 1.1 游戏名

《我只要一刀》

### 1.2 一句话玩法

敌军压境，玩家蓄势挥刀；刀势越满，一刀越强；看准阵眼，一刀破阵，满屏溃散。

### 1.3 核心体验

玩家不控制角色移动，不点多个技能，不做复杂组合操作。玩家核心动作仍然只有一个：

> 按下并拖动，挥出一刀。

但 V0.2 中“一刀”不再是固定冷却后的单条直线，而是一次短暂、可衰减的 **刀芒轨迹**：

```text
敌军压近
↓
刀势随时间自动恢复
↓
玩家观察刀势强弱、敌阵结构、补给位置
↓
随时可以按下挥刀
↓
指尖刀芒实时切割敌军
↓
刀芒随时间 / 距离 / 急转弯逐渐变弱
↓
松手或刀芒耗尽，武将收刀
↓
统一结算火药爆炸 / 阵眼崩散 / 连锁评价
↓
击杀与连锁反哺刀势恢复
```

玩家需要理解的核心规则只有一句：

> 刀亮不亮，决定这一刀强不强；随时能砍，但强刀更爽。

### 1.4 包装方向

采用 **敌军 / 兵阵 / 纸片兵** 包装。

敌人不是写实血肉士兵，而是：

- 纸片兵；
- 兵牌；
- 木偶兵；
- 影子兵；
- 符纸军阵。

底部角色采用 **城墙守阵武将** 表现：

- 武将站在城墙上；
- 刀势越高，武将刀鞘 / 刀身越亮；
- 玩家按下时，武将瞬间拔刀；
- 玩家拖动时，指尖刀气立即切割敌阵；
- 武将不位移，不飞到战场中，避免延迟感；
- 松手或刀芒耗尽时，武将收刀并触发连锁爆发。

### 1.5 V0.2 相对 V0.1 的核心变化

| 模块 | V0.1 | V0.2 |
|---|---|---|
| 挥刀节奏 | 每隔固定冷却获得一次斩击机会 | 玩家随时可挥刀，强度由刀势能量决定 |
| 大招表现 | 可能存在独立斩意大招 | 不设置独立大招按钮，满刀势自然就是破阵斩 |
| 一刀边界 | 按下拖动，松手后生成一条线 | 按下后实时产生刀芒，刀芒逐渐变弱直到消失 |
| 命中方式 | 松手后一次性判定 | 拖动过程实时命中，收刀时统一结算连锁 |
| UI 表现 | 底部冷却条 | 底部武将蓄势 + 指尖刀芒强弱 + 战斗区轻反馈 |
| 体验目标 | 等待敌人密集后切 | 随时能切，但等待可换取更强、更长、更爽的一刀 |

## 2. 原型版本目标

### 2.1 V0.2 必须验证的问题

1. 取消固定冷却后，“随时可挥刀但强刀更爽”是否能降低等待疲劳？
2. 玩家能否直观理解刀势能量与攻击强度的关系？
3. 指尖刀芒逐渐变弱，是否能自然传达“一刀”的时长和距离边界？
4. 实时切割是否比松手结算更有手感？
5. 收刀后统一触发连锁爆发，是否能保留“一刀破阵”的完整爽感？
6. 底部武将只做同步表现、不参与位移，是否能解决操作和场景表现的衔接问题？
7. 随机补给是否能在不增加复杂操作的前提下提升单局变化？
8. 主线闯关是否能继续承载图鉴和装备目标？

### 2.2 V0.2 范围

V0.2 仍然只做主线闯关原型，不做完整商业化，不做复杂养成。

## 5. 单局核心规则

### 5.1 胜利条件

清完当前关卡所有波次，并且玩家生命大于 0。

### 5.2 失败条件

敌军触碰底部防线扣除生命。  
生命归零后立即失败。

### 5.3 单局时长

V0.2 每关建议 45-90 秒。

| 关卡阶段 | 建议时长 |
|---|---:|
| 第 1-3 关 | 30-45 秒 |
| 第 4-7 关 | 45-70 秒 |
| 第 8-10 关 | 70-90 秒 |

### 5.4 防线规则

防线位于画布底部上方：

```ts
const DEFENSE_LINE_Y = 720;
```

敌人中心点 y 坐标超过防线，即视为触线。

触线后：

1. 敌人从场上移除；
2. 玩家生命扣除对应数值；
3. 触发红色屏幕闪烁；
4. 触发生命变化动画；
5. 如果生命 <= 0，进入失败结算。

### 5.5 玩家生命

V0.2 初始生命：

```ts
player.hp = 3;
```

不同敌人触线伤害：

| 敌军 | 触线伤害 |
|---|---:|
| 纸片步兵 | 1 |
| 盾兵 | 1 |
| 火药兵 | 1 |
| 阵眼兵 | 2 |

## 6. 玩家操作规则 V0.2

### 6.1 唯一操作：按下并拖动，挥出一刀

玩家随时可以挥刀。  
没有固定冷却时间。  
本次挥刀强度由按下瞬间的刀势能量决定。

### 6.2 刀势能量 Blade Energy

刀势能量是 V0.2 唯一的攻击资源。

```ts
interface BladeEnergyState {
  value: number;       // 0-100
  max: number;         // 100
  regenPerSecond: number;
  killGain: number;
  chainGain: number;
  isLockedForCurrentSlash: boolean;
}
```

基础规则：

- 刀势范围 0-100；
- 不挥刀时随时间自动恢复；
- 击杀敌军会获得少量刀势；
- 连锁、火药爆炸、阵眼崩散会获得额外刀势；
- 玩家按下时锁定本刀等级；
- 拖动过程中消耗刀势对应的“刀芒耐久”；
- 一刀结束后，继续从当前能量状态恢复。

### 6.3 刀势阶段

```ts
export type BladeTier = 'weak' | 'normal' | 'strong' | 'burst';
```

| 刀势区间 | 阶段 | 视觉表现 | 实际效果 | 玩家理解 |
|---:|---|---|---|---|
| 0-25 | weak 残锋 | 暗淡、短、细 | 可应急切步兵，距离短 | 能砍，但很弱 |
| 25-60 | normal 常锋 | 白金刀芒 | 标准伤害，正常范围 | 常规一刀 |
| 60-90 | strong 强锋 | 金色、更宽、更长 | 可破盾，连锁增强 | 强力一刀 |
| 90-100 | burst 破阵锋 | 炽金、符文、震动 | 破阵斩，火药/阵眼收益大 | 这一刀很爽 |

注意：

- 不要再做独立“大招按钮”；
- 满刀势不是另一个技能，而是让下一刀自然变成“破阵锋”；
- 玩家要理解的是“刀越亮越强”，不是“等 CD + 等大招”。

### 6.4 刀势恢复

建议数值：

```ts
const BLADE_ENERGY_MAX = 100;
const BLADE_ENERGY_REGEN_PER_SECOND = 16; // 约 6.25 秒从 0 到满
```

击杀回能建议：

| 行为 | 刀势恢复 |
|---|---:|
| 切中步兵 | +1 |
| 击杀步兵 | +2 |
| 击杀盾兵 | +4 |
| 切中火药兵 | +4 |
| 火药连锁击杀 | +2 / 个 |
| 切中阵眼 | +8 |
| 阵眼崩散击杀 | +1 / 个 |
| 一刀击杀 10 个以上 | +10 |

上限限制：

```ts
bladeEnergy.value = Math.min(100, bladeEnergy.value + gain);
```

### 6.5 一刀的边界定义

V0.2 中，一刀定义为：

> 从玩家按下屏幕开始，到玩家松手或刀芒耗尽结束的一次完整斩击轨迹。

不是无限滑动，也不是松手后才攻击。

### 6.6 斩击状态机

```ts
export type SlashState = 'idle' | 'slashing' | 'settling';
```

流程：

```text
idle
↓ 玩家按下
slashing：实时切割
↓ 松手 / 刀芒耗尽
settling：收刀结算连锁
↓ 连锁表现结束
idle
```

### 6.7 SlashTrail 数据结构

V0.2 不再只保存一条直线，而保存一段轨迹。

```ts
interface SlashPoint {
  x: number;
  y: number;
  t: number;
  energyRatio: number; // 当前刀芒剩余比例 0-1
}

interface SlashTrail {
  id: string;
  startedAt: number;
  endedAt?: number;

  startEnergy: number;
  tier: BladeTier;

  remainingPower: number;  // 本刀刀芒耐久，拖动时衰减
  maxPower: number;

  points: SlashPoint[];
  hitEnemyIds: Set<string>;
  hitSupplyIds: Set<string>;

  pendingExplosions: string[];
  pendingCoreCollapses: string[];

  killCount: number;
  chainCount: number;
  isActive: boolean;
}
```

### 6.8 刀芒衰减规则

刀芒衰减来自三部分：

1. 时间消耗；
2. 距离消耗；
3. 急转弯惩罚。

建议参数：

```ts
const SLASH_POWER_BY_TIER = {
  weak: 70,
  normal: 130,
  strong: 210,
  burst: 300
};

const SLASH_TIME_COST_PER_SECOND = 45;
const SLASH_DISTANCE_COST_PER_PX = 0.55;
const SLASH_SHARP_TURN_EXTRA_COST = 18;
const SLASH_SHARP_TURN_ANGLE_DEG = 85;
```

每次 pointer move：

```ts
const dt = now - lastPoint.t;
const dist = distance(lastPoint, currentPoint);
let cost = dt * SLASH_TIME_COST_PER_SECOND + dist * SLASH_DISTANCE_COST_PER_PX;

if (isSharpTurn(prevPoint, lastPoint, currentPoint)) {
  cost += SLASH_SHARP_TURN_EXTRA_COST;
}

trail.remainingPower -= cost;
```

当 `remainingPower <= 0`，本刀结束。

### 6.9 同一敌人每刀最多命中一次

必须实现，防止玩家在同一敌人身上来回刷伤害。

```ts
if (trail.hitEnemyIds.has(enemy.id)) return;
trail.hitEnemyIds.add(enemy.id);
damageEnemy(enemy, slashDamage);
```

### 6.10 低能量也能挥刀，但收益低

低刀势挥刀不能被禁止。  
否则会重新变成等待冷却。

低刀势的意义是：

- 应急保命；
- 切补给；
- 补刀；
- 清少量近处敌人。

高刀势的意义是：

- 一刀贯穿；
- 破盾；
- 切阵眼；
- 触发火药连锁；
- 打出“一刀破阵”。

## 7. 攻击结算规则：实时切割 + 收刀结算

### 7.1 为什么不用“画完线再攻击”

画完线再攻击会导致：

- 玩家划过敌人时没有即时反馈；
- 敌人在移动，松手结算可能导致判定不准；
- 手感偏慢，不适合轻休闲爽感。

### 7.2 推荐方案

采用：

> 划动过程实时命中，松手 / 刀芒耗尽时统一触发连锁爆发。

### 7.3 详细流程

```text
玩家按下
↓
读取当前刀势，确定本刀 tier 和 maxPower
↓
底部武将瞬间拔刀
↓
指尖生成刀芒
↓
玩家拖动
↓
刀芒经过敌军，实时命中与伤害
↓
普通步兵可即时碎裂
↓
盾兵即时破盾 / 死亡
↓
火药兵被切中后先标记为“点燃”
↓
阵眼兵被切中后先标记为“裂阵”
↓
玩家松手或刀芒耗尽
↓
武将收刀
↓
统一触发火药爆炸、阵眼崩散、连锁播报
```

### 7.4 普通命中实时反馈

实时命中时立即反馈：

- 敌人闪白；
- 切痕出现；
- 纸屑粒子；
- 小音效；
- 分数轻飘字。

### 7.5 连锁延迟到收刀触发

以下内容建议在收刀阶段统一触发：

- 火药兵爆炸；
- 火药连锁；
- 阵眼崩散；
- 同阵连锁；
- 一刀评价；
- 慢镜头 / 震屏。

这样不会在拖动过程中让画面过乱，也能形成“一刀收势后全阵崩散”的高潮。

## 8. 刀势表现与 UI 规范

刀势强关联攻击强度，因此必须重点设计。  
不建议只做一个传统进度条。  
推荐使用三层表现：

```text
底部武将 / 刀鞘蓄势
+
指尖刀芒强弱
+
战斗区轻反馈
```

### 8.1 第一层：底部武将 / 刀鞘蓄势

底部武将是刀势的主视觉表现。

| 刀势阶段 | 底部表现 |
|---|---|
| 0-25 | 武将刀身暗，只有微光 |
| 25-60 | 刀身白金光，普通蓄势姿态 |
| 60-90 | 刀身金光明显，武将压低重心 |
| 90-100 | 刀身炽金，城墙周围有气浪与符文 |

玩家看到底部武将状态，就能判断：

> 现在这一刀强不强。

### 8.2 第二层：指尖刀芒强度

玩家按下后，真正操作位置在手指，因此指尖刀芒必须传达强度和剩余边界。

| 刀势阶段 | 指尖刀芒表现 |
|---|---|
| weak | 短、细、暗，拖尾短 |
| normal | 白金刀光，标准宽度 |
| strong | 金色刀光，更宽，拖尾明显 |
| burst | 炽金刀光，边缘带符文、屏幕轻震 |

拖动过程中：

- 刀芒逐渐变细；
- 颜色由亮变暗；
- 尾迹逐渐断裂；
- 快耗尽时发出闪烁；
- 耗尽后立即收刀。

### 8.3 第三层：战斗区轻反馈

战斗区不要堆 UI，只做轻提示：

- 刀势达到 strong 时，盾兵盾牌边缘微微发亮，提示可破盾；
- 刀势达到 burst 时，阵眼兵出现共鸣光环，提示可破阵；
- 满刀势时，屏幕边缘出现短暂金色呼吸；
- 切中火药兵时出现“点燃”小标记。

### 8.4 战斗 HUD

战斗界面元素：

| 位置 | 内容 |
|---|---|
| 顶部左侧 | 生命：❤❤❤ |
| 顶部中间 | 当前波次：3/6 |
| 顶部右侧 | 分数 |
| 底部上方 | 防线 |
| 底部中间 | 城墙武将 + 刀势蓄势表现 |
| 指尖位置 | 刀芒强弱与剩余边界 |
| 屏幕中央 | 斩击评价浮字 |

### 8.5 不要同时展示两个进度

V0.2 禁止同时展示：

- 普通挥刀冷却；
- 大招能量；
- 另一个斩意进度。

只允许一个核心资源：

> 刀势能量。

### 8.6 可选的轻量能量条

如果仅靠武将发光不够清晰，可以在底部武将脚下增加一个非常短的半弧形刀气槽。

要求：

- 不要像传统冷却条；
- 不要大面积占屏；
- 颜色和武将刀光同步；
- 文案最多显示“残锋 / 常锋 / 强锋 / 破阵锋”，不要显示复杂数字。

## 9. 底部武将与操作衔接

### 9.1 核心原则

底部武将不飞入战场。  
玩家控制的是远程刀气。

### 9.2 正确表现

```text
玩家按下任意战斗区域
↓
指尖立刻出现刀芒并可命中敌人
↓
底部武将同步拔刀
↓
玩家拖动，刀芒实时切割
↓
武将动作方向轻微跟随玩家划线方向
↓
松手 / 刀芒耗尽，武将收刀
↓
触发连锁崩散
```

### 9.3 不建议表现

不要做成：

```text
玩家划线
↓
武将跳到上方
↓
武将再挥刀
↓
敌人才受伤
```

原因：

- 反馈延迟；
- 操作和结果脱节；
- 轻休闲手感变差。

### 9.4 气脉连接表现

可以在玩家按下瞬间，从底部武将到指尖闪一下极短的金色气脉。

要求：

- 持续 0.08-0.12 秒；
- 只是视觉连接；
- 不承担真实飞行弹道；
- 不影响命中判定。

作用：

> 让玩家理解指尖刀气来自底部武将。

## 10. 随机补给系统 V0.2

### 10.1 设计目标

随机补给用于缓解单局疲劳，增加复玩变化。  
玩家不需要点按钮，不需要选择技能。  
补给直接出现在战场中，玩家用刀芒切过即可获得。

### 10.2 补给数据结构

```ts
export type SupplyType = 'blade_soul' | 'fire_oil' | 'war_drum';

export interface SupplyItem {
  id: string;
  type: SupplyType;
  x: number;
  y: number;
  radius: number;
  active: boolean;
  expiresAt?: number;
}
```

### 10.3 V0.2 必做 3 种补给

| 补给 | 效果 | 持续方式 |
|---|---|---|
| 刀魂 blade_soul | 下一刀 maxPower +25%，刀芒更长 | 影响下一刀 |
| 火油 fire_oil | 下一刀命中敌军后附带小范围灼爆 | 影响下一刀 |
| 战鼓 war_drum | 5 秒内刀势恢复速度 +50% | 限时状态 |

### 10.4 补给出现规则

V0.2 建议：

- 每波最多出现 1 个补给；
- 前 3 关不出现补给；
- 第 4 关开始出现；
- 补给出现位置不能太靠近防线；
- 补给存活 6-8 秒后消失；
- 补给不能挡住阵眼和火药兵阅读。

伪代码：

```ts
function maybeSpawnSupply(wave: WaveConfig) {
  if (currentLevel.id < 4) return;
  if (Math.random() > 0.35) return;
  spawnRandomSupplyNearFormation(wave.formations[0]);
}
```

### 10.5 补给交互

补给和敌军一样参与刀芒轨迹检测。

```ts
if (distanceSegmentToPoint(segment, supply.position) <= supply.radius) {
  collectSupply(supply);
}
```

补给被切中后：

- 播放吸收粒子；
- 显示简短文案，例如“刀魂入锋”；
- 不打断当前挥刀；
- 效果进入 `player.buffs`。

### 11.2 V0.2 敌军类型

#### 11.2.1 纸片步兵 paper_soldier

基础敌人，一刀即碎。

| 参数 | 数值 |
|---|---:|
| hp | 1 |
| radius | 14 |
| speed | 18-28 px/s |
| touchDamage | 1 |
| scoreValue | 10 |
| chainTags | normal |

#### 11.2.2 盾兵 shield_soldier

前排抗压单位。

| 参数 | 数值 |
|---|---:|
| hp | 2 |
| radius | 16 |
| speed | 15-24 px/s |
| touchDamage | 1 |
| scoreValue | 25 |
| chainTags | shield |

规则：

- weak / normal 阶段：每次命中造成 1 点伤害；
- strong / burst 阶段：可以额外破盾，造成 2 点伤害；
- 被爆炸命中也会扣 1 点血；
- 阵眼崩散可以直接击杀。

#### 11.2.3 火药兵 powder_soldier

连锁爆炸核心。

| 参数 | 数值 |
|---|---:|
| hp | 1 |
| radius | 15 |
| speed | 17-26 px/s |
| touchDamage | 1 |
| scoreValue | 30 |
| chainTags | explosive |

V0.2 规则：

- 被刀芒命中时立即死亡，并标记 pending explosion；
- 拖动过程中先显示“点燃”反馈；
- 收刀阶段触发爆炸；
- 爆炸半径内敌人受到 1 点伤害；
- 如果爆炸击杀另一个火药兵，会继续触发爆炸；
- 连锁次数上限建议 8，防止无限循环。

爆炸参数：

```ts
const EXPLOSION_RADIUS = 58;
const EXPLOSION_DAMAGE = 1;
const MAX_CHAIN_DEPTH = 8;
```

#### 11.2.4 阵眼兵 formation_core

一刀破阵核心。

| 参数 | 数值 |
|---|---:|
| hp | 1 |
| radius | 17 |
| speed | 14-22 px/s |
| touchDamage | 2 |
| scoreValue | 50 |
| chainTags | core |

V0.2 规则：

- 被刀芒命中时立即死亡，并标记 pending core collapse；
- 拖动过程中先显示“裂阵”反馈；
- 收刀阶段触发同 formationId 敌军崩散；
- 崩散最多影响 10 个敌人；
- 被崩散击杀的敌人计入连锁击杀；
- 崩散不能再次触发阵眼效果，防止链式全屏过强。

崩散参数：

```ts
const CORE_COLLAPSE_MAX_TARGETS = 10;
```

## 15. 命中、伤害与连锁

### 15.1 实时命中检测

每次 `pointermove`，系统只检测新增轨迹段。

```ts
const segment = {
  a: previousPoint,
  b: currentPoint
};

for (const enemy of aliveEnemies) {
  if (trail.hitEnemyIds.has(enemy.id)) continue;

  const hit = distancePointToSegment(enemy.position, segment.a, segment.b) <= getSlashHitWidth(trail);

  if (hit) {
    trail.hitEnemyIds.add(enemy.id);
    applySlashHit(enemy, trail);
  }
}
```

### 15.2 命中宽度

命中宽度随刀势阶段和剩余刀芒变化。

```ts
function getSlashHitWidth(trail: SlashTrail) {
  const base = {
    weak: 8,
    normal: 12,
    strong: 16,
    burst: 22
  }[trail.tier];

  const ratio = Math.max(0.35, trail.remainingPower / trail.maxPower);
  return base * ratio;
}
```

### 15.3 伤害规则

```ts
function getSlashDamage(trail: SlashTrail, enemy: Enemy) {
  if (trail.tier === 'weak') return 1;
  if (trail.tier === 'normal') return 1;
  if (trail.tier === 'strong') return enemy.type === 'shield_soldier' ? 2 : 1;
  if (trail.tier === 'burst') return enemy.type === 'shield_soldier' ? 2 : 1;
  return 1;
}
```

对不同敌军：

| 敌军 | 实时命中效果 | 收刀结算效果 |
|---|---|---|
| 纸片步兵 | 直接击杀 | 无 |
| 盾兵 | 扣血 / 破盾 / 死亡 | 无 |
| 火药兵 | 死亡并标记点燃 | 爆炸连锁 |
| 阵眼兵 | 死亡并标记裂阵 | 同阵崩散 |

### 15.4 收刀结算流程

```text
收刀开始
↓
按命中顺序处理 pendingExplosions
↓
处理爆炸范围伤害与火药连锁
↓
按命中顺序处理 pendingCoreCollapses
↓
处理同阵敌军崩散
↓
统计本刀击杀数、最大连锁、得分
↓
根据表现增加刀势回能
↓
播放评价浮字
↓
退出 slash_settling
```

### 15.5 火药爆炸结算

```ts
function explode(sourceEnemy: Enemy, depth: number) {
  if (depth > MAX_CHAIN_DEPTH) return;

  createExplosionEffect(sourceEnemy.x, sourceEnemy.y);

  for (const enemy of aliveEnemies) {
    const d = distance(sourceEnemy, enemy);
    if (d <= EXPLOSION_RADIUS) {
      damageEnemy(enemy, EXPLOSION_DAMAGE);

      if (!enemy.isAlive && enemy.type === 'powder_soldier') {
        explode(enemy, depth + 1);
      }
    }
  }
}
```

### 15.6 阵眼崩散结算

```ts
function collapseFormation(coreEnemy: Enemy) {
  const targets = aliveEnemies
    .filter(e => e.formationId === coreEnemy.formationId)
    .filter(e => e.id !== coreEnemy.id)
    .slice(0, CORE_COLLAPSE_MAX_TARGETS);

  for (const target of targets) {
    killEnemy(target, 'formation_collapse');
    createCollapseEffect(target.x, target.y);
  }
}
```

规则：

- 只影响同 formationId；
- 最多影响 10 个目标；
- 被崩散击杀的火药兵默认不再触发爆炸，避免过强；
- 如果后续要增强爽感，可通过装备允许“阵眼崩散触发火药”。

## 24. 关键算法说明

### 24.1 线段到点距离

用于判断新刀芒轨迹段是否命中圆形敌人或补给。

```ts
export function distancePointToSegment(p: Vec2, a: Vec2, b: Vec2): number {
  const ab = { x: b.x - a.x, y: b.y - a.y };
  const ap = { x: p.x - a.x, y: p.y - a.y };

  const abLenSq = ab.x * ab.x + ab.y * ab.y;
  if (abLenSq === 0) {
    return Math.hypot(p.x - a.x, p.y - a.y);
  }

  let t = (ap.x * ab.x + ap.y * ab.y) / abLenSq;
  t = Math.max(0, Math.min(1, t));

  const closest = {
    x: a.x + ab.x * t,
    y: a.y + ab.y * t
  };

  return Math.hypot(p.x - closest.x, p.y - closest.y);
}
```

### 24.2 急转弯判断

用于限制玩家无限乱划。

```ts
function angleBetween(a: Vec2, b: Vec2): number {
  const dot = a.x * b.x + a.y * b.y;
  const la = Math.hypot(a.x, a.y);
  const lb = Math.hypot(b.x, b.y);
  if (la === 0 || lb === 0) return 0;
  const cos = Math.max(-1, Math.min(1, dot / (la * lb)));
  return Math.acos(cos) * 180 / Math.PI;
}

function isSharpTurn(p0: Vec2, p1: Vec2, p2: Vec2): boolean {
  const v1 = { x: p1.x - p0.x, y: p1.y - p0.y };
  const v2 = { x: p2.x - p1.x, y: p2.y - p1.y };
  return angleBetween(v1, v2) > TUNING.slashSharpTurnAngleDeg;
}
```

```ts
function getBladeTier(value: number): BladeTier {
  if (value < 25) return 'weak';
  if (value < 60) return 'normal';
```


### 24.5 更新一刀

```ts
function updateSlash(pos: Vec2, now: number) {
  if (!currentTrail || !currentTrail.isActive) return;

  const points = currentTrail.points;
  const last = points[points.length - 1];
  const prev = points.length >= 2 ? points[points.length - 2] : null;

  const dt = (now - last.t) / 1000;
  const dist = distance(last, pos);

  let cost = dt * TUNING.slashTimeCostPerSecond + dist * TUNING.slashDistanceCostPerPx;

  if (prev && isSharpTurn(prev, last, pos)) {
    cost += TUNING.slashSharpTurnExtraCost;
  }

  currentTrail.remainingPower -= cost;

  const ratio = Math.max(0, currentTrail.remainingPower / currentTrail.maxPower);

  const newPoint = { x: pos.x, y: pos.y, t: now, energyRatio: ratio };
  points.push(newPoint);

  checkSegmentHits(last, newPoint, currentTrail);

  if (currentTrail.remainingPower <= 0) {
    endSlash('power_empty');
  }
}
```

> 玩家第一次看到一刀收势后爆炸清场。

### 25.5 第 5 关：一刀破阵

目标：引入阵眼兵。

配置：

- 每个阵型带一个阵眼兵；
- 切普通兵收益一般；
- 切阵眼兵，收刀后全阵崩散。

体验：

> 玩家理解“找阵眼，而不是切最多”。

### 25.6 第 6 关：盾墙压境

目标：引入盾兵和 strong 阶段价值。

配置：

- 前排盾兵；
- 后排纸片步兵；
- weak / normal 阶段切盾收益低；
- strong / burst 阶段破盾明显。

体验：

> 玩家理解强刀可以解决硬目标。

### 25.7 第 7 关：火药藏后

目标：组合盾兵和火药兵。

配置：

- 前排盾兵；
- 后排火药兵；
- 玩家需要斜切绕过盾兵切火药兵；
- 可通过火油补给增强爆炸。

体验：

> 玩家学会选择后排高价值目标。

### 25.8 第 8 关：阵眼护卫

目标：增加切线规划。

配置：

- 阵眼兵被普通兵和盾兵保护；
- 只有斜向角度能切中阵眼；
- 刀芒过度急转会更快衰减。

体验：

> 玩家感觉自己是在破阵。

### 25.9 第 9 关：连环乱阵

目标：综合测试。

配置：

- 横列阵、斜锋阵、密集阵混合；
- 火药兵和阵眼兵同时出现；
- 敌人速度提高；
- 补给随机出现。

体验：

> 玩家在一局内连续做“应急弱刀 / 等强刀 / 切补给 / 破阵”的决策。

### 25.10 第 10 关：黄巾大阵

目标：小 Boss 阵。

配置：

- 3 个阶段；
- 第一阶段盾墙；
- 第二阶段火药连锁；
- 第三阶段阵眼核心；
- 通关后必掉一件稀有装备。

体验：

> 玩家用前面学到的所有规则完成章节挑战。

## 28. 验收标准

### 28.1 核心玩法验收

必须满足：

- 玩家只需要按下拖动即可完成主要操作；
- 没有固定斩击冷却；
- 玩家随时可以挥刀；
- 刀势越高，刀芒越长、越亮、越强；
- 低刀势挥刀可以应急，但收益明显低；
- 拖动过程中实时命中敌军；
- 刀芒会随时间、距离、急转弯逐渐变弱；
- 同一敌人每刀最多命中一次；
- 松手或刀芒耗尽后收刀结算连锁；
- 火药兵可以连锁爆炸；
- 阵眼兵可以让同阵敌人崩散；
- 底部武将不位移，但能与玩家操作同步；
- 敌人压近防线带来紧张感；
- 玩家愿意为了更强一刀等待刀势恢复。

### 28.2 刀势传达验收

必须满足：

- 不看数字也能感知刀势大致强弱；
- 底部武将蓄势状态能表达刀势积累；
- 指尖刀芒能表达本次挥刀强度；
- 刀芒衰减能表达本次一刀即将结束；
- strong / burst 阶段有明显视觉差异；
- 不出现“冷却条 + 大招条”两个进度。

### 28.3 系统完整性验收

必须满足：

- 有主菜单；
- 有关卡选择；
- 有 10 个关卡；
- 有胜利和失败；
- 有结算评级；
- 有图鉴解锁；
- 有装备掉落和装备生效；
- 有 3 种补给；
- 有 localStorage 保存进度；
- 刷新页面后进度不丢失。

### 28.4 移动端验收

必须满足：

- 手机竖屏打开不变形；
- 触控拖动顺滑；
- 不出现页面滚动影响操作；
- 指尖刀芒跟手；
- 按钮可点击；
- 战斗区域适配屏幕。

## 30. 设计底线

### 30.1 不增加复杂操作

不要加入：

- 角色移动；
- 多技能按钮；
- 普通攻击按钮；
- 复杂走位；
- 多手势组合。

玩家核心动作永远是：

> 按下并拖动，挥出一刀。

### 30.2 不出现双进度等待

禁止让玩家同时等：

- 普通挥刀冷却；
- 大招能量；
- 额外技能蓄力。

只保留：

> 刀势能量。

### 30.3 装备只强化这一刀

装备不要变成传统 RPG 属性堆叠。  
装备应该影响：

- 刀芒更长；
- 刀势恢复更快；
- 击杀回能更多；
- 爆炸更大；
- 破盾更强；
- 阵眼更爽；
- 防线更稳。

### 30.4 关卡像“破阵题”

关卡不要只是堆怪。  
每一关都应该让玩家思考：

> 现在用弱刀保命，还是再等一秒打强刀？  
> 这一刀从哪里下？  
> 要不要顺手切补给？  
> 先切火药，还是先切阵眼？

## 31. 最小可玩版本优先级

如果开发时间非常有限，按以下顺序做：

1. Canvas 战斗场景；
2. 敌人下落；
3. 刀势能量自动恢复；
4. 玩家按下拖动生成刀芒轨迹；
5. 刀芒实时命中敌人；
6. 刀芒衰减并结束一刀；
7. 底部武将蓄势 / 拔刀 / 收刀表现；
8. 火药兵收刀爆炸；
9. 阵眼兵收刀崩散；
10. 生命和防线；
11. 3 个关卡；
12. 结算评分；
13. 随机补给；
14. 扩到 10 个关卡；
15. 图鉴；
16. 装备。

前三天只追求一件事：

> 玩家看到刀势变亮，按下拖动，刀芒实时切开敌军，收刀后敌阵爆开一片。

只要这个爽感成立，后续图鉴、装备、关卡、无尽才有意义。
